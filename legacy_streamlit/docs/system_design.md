# 系统设计

## 系统定位

HEYIYUN 中国电力工程企业项目经营管理平台面向电力工程企业，围绕项目全生命周期开展经营管理，覆盖立项、报装、合同、采购、施工、成本、资金、安全、质量、资料、审批和风险预警。

## 技术架构

- 表现层：Streamlit 多页面。
- 组件层：`components/` 封装指标卡、表格、图表、风险标签、流程轨迹和详情 Tabs。
- 服务层：`services/` 承载项目、合同、成本、资金、风险、审批、权限等业务逻辑。
- 数据层：`data/` 生成强关联 mock 数据。
- 工具层：`utils/` 提供格式化、导出、session state 和校验能力。

## 模块划分

系统划分为首页工作台、决策驾驶舱、项目中心、合同中心、采购物资中心、设备周材中心、成本资金中心、施工现场中心、安全质量中心、资料档案中心、审批中心、风险预警中心、逻辑图中心和系统设置。

## 页面结构

入口为 `app.py`，其他页面位于 `pages/`。页面层只负责筛选、展示和交互，业务计算统一调用 `services/`。

## 数据流

项目主数据通过 `project_id` 关联合同、成本、资金、材料、设备、进度、安全质量、资料、审批和风险。经营指标先从 mock 数据汇总，再由组件层渲染为 KPI、表格和图表。

## 权限模型

角色保存在 `st.session_state["current_role"]`。`permission_service.py` 负责模块可见性、金额脱敏和审批权限判断。
